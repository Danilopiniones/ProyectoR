Introduccion

El mercado de criptomonedas, y en particular el Bitcoin, se caracteriza por su alta volatilidad y sensibilidad a factores externos, entre ellos, la opini√≥n p√∫blica expresada en redes sociales. Este proyecto tiene como objetivo principal predecir el precio diario de cierre del Bitcoin utilizando an√°lisis de sentimiento extra√≠do de tweets relacionados con la criptomoneda durante el a√±o 2021.

Para ello, se aplican t√©cnicas de procesamiento de lenguaje natural para cuantificar el sentimiento diario, que luego se combina con datos hist√≥ricos de precios y volumen para construir un modelo predictivo basado en Random Forest. Este modelo busca capturar patrones tanto en la evoluci√≥n del precio como en las se√±ales emocionales del mercado para mejorar la precisi√≥n en la predicci√≥n.

Adem√°s, se dise√±an y comparan distintas estrategias de inversi√≥n basadas en las predicciones generadas, evaluando su desempe√±o en t√©rminos de rentabilidad y riesgo. Esto permite no solo validar la utilidad pr√°ctica del modelo, sino tambi√©n aportar ideas para la toma de decisiones informadas en un mercado altamente din√°mico y complejo. # Objetivos del Proyecto

El objetivo de este proyecto es predecir el precio diario de cierre del Bitcoin en base al an√°lisis de sentimiento de publicaciones en Twitter, utilizando modelos de aprendizaje autom√°tico como Random Forest. Adem√°s, se simulan estrategias de inversi√≥n basadas en las predicciones obtenidas.

4. Creacion de variables

En esta secci√≥n se realiza la creaci√≥n de variables para enriquecer el conjunto de datos con caracter√≠sticas relevantes que permitan mejorar la capacidad predictiva del modelo. Se generan variables rezagadas (lags) de las principales m√©tricas, como el sentimiento promedio, volumen y precios de apertura, para capturar la influencia del comportamiento hist√≥rico en el precio diario del Bitcoin. Adem√°s, se calculan indicadores como el retorno absoluto y el retorno porcentual diario para reflejar las variaciones del precio durante la jornada. Se incluye tambi√©n la variable categ√≥rica del d√≠a de la semana para incorporar posibles efectos estacionales o patrones asociados al calendario. Finalmente, se calcula la volatilidad acumulada, que representa la variabilidad hist√≥rica de los retornos y es fundamental para entender la incertidumbre del mercado. Estas transformaciones permiten que el modelo tenga en cuenta tanto informaci√≥n hist√≥rica como patrones temporales y de riesgo, mejorando as√≠ su desempe√±o predictivo.

5. Modelo Random Forest con validaci√≥n cruzada

El modelo utilizado para predecir el precio de cierre diario del Bitcoin es del tipo Random Forest, un algoritmo de aprendizaje autom√°tico basado en √°rboles de decisi√≥n que combina m√∫ltiples modelos para mejorar la precisi√≥n y reducir el sobreajuste. Para realizar este modelo, se seleccionaron variables que reflejan tanto el comportamiento hist√≥rico del mercado como el sentimiento expresado en redes sociales. Entre las variables principales se encuentran el precio de apertura del d√≠a y sus valores rezagados a 1, 2 y 3 d√≠as, que permiten captar tendencias recientes en el mercado. Adem√°s, se incorporan las medidas de sentimiento promedio de tweets relacionados con Bitcoin, tambi√©n con rezagos de hasta tres d√≠as, para incluir la influencia del estado emocional de los inversores. Finalmente, se incluyen variables de volumen de transacciones rezagadas, que aportan informaci√≥n sobre la actividad del mercado. La combinaci√≥n de estas variables busca proporcionar al modelo una visi√≥n integral que facilite la predicci√≥n del precio de cierre con mayor precisi√≥n.

9. Comparaci√≥n de estrategias de inversi√≥n

En esta secci√≥n se eval√∫an distintas estrategias de inversi√≥n simuladas utilizando las predicciones generadas por el modelo. Se comparan alternativas basadas en se√±ales de compra y venta derivadas del an√°lisis de sentimiento y la evoluci√≥n del precio del Bitcoin. El objetivo es analizar cu√°l estrategia ofrece un mejor rendimiento ajustado al riesgo, considerando m√©tricas como la rentabilidad acumulada, la volatilidad y el drawdown m√°ximo. Esta comparaci√≥n permite validar el valor pr√°ctico del modelo predictivo no solo en t√©rminos estad√≠sticos, sino tambi√©n en su aplicaci√≥n financiera real, facilitando la toma de decisiones informadas para invertir en mercados vol√°tiles como el de las criptomonedas.

9.1 Estrategia 1

Esta estrategia consiste en invertir siempre que el modelo prediga una subida en el precio de Bitcoin. Es decir, si el modelo estima que el precio de cierre ser√° mayor que el precio de apertura, se realiza la inversi√≥n ese d√≠a. Esta aproximaci√≥n es sencilla y busca aprovechar las se√±ales positivas generadas por el modelo para obtener rentabilidad. La idea es capitalizar las oportunidades en las que el modelo anticipa un movimiento alcista, evitando invertir en d√≠as con predicciones negativas o neutras.

9.2 Estrategia 2

La segunda estrategia es una variaci√≥n de la primera, en la que se evita invertir en los d√≠as martes y jueves, aun cuando la predicci√≥n sea positiva. Esta decisi√≥n se basa en la observaci√≥n de que ciertos d√≠as de la semana pueden presentar comportamientos menos favorables o mayor volatilidad, lo que podr√≠a aumentar el riesgo. Por tanto, se filtran esos d√≠as espec√≠ficos para tratar de mejorar el rendimiento y reducir p√©rdidas potenciales, limitando la inversi√≥n solo a los d√≠as con mayor probabilidad de √©xito seg√∫n el modelo.

9.3 Estrategia 3

Se introduce un filtro m√°s exigente para decidir cu√°ndo invertir, bas√°ndose en un umbral fijo que se calcula a partir de la desviaci√≥n est√°ndar hist√≥rica de los retornos reales. Solo se realiza la inversi√≥n cuando la predicci√≥n de subida supera la mitad de esta desviaci√≥n est√°ndar, es decir, cuando el modelo anticipa un cambio significativo en el precio. Esta t√°ctica busca minimizar las inversiones en movimientos peque√±os o poco confiables, enfoc√°ndose en las se√±ales m√°s fuertes para incrementar la probabilidad de obtener ganancias sustanciales.

9.4 Estrategia 4

se utiliza un umbral adaptativo que var√≠a din√°micamente seg√∫n la volatilidad acumulada del mercado. Este enfoque ajusta la exigencia para invertir dependiendo de c√≥mo cambia la volatilidad a lo largo del tiempo: en periodos de alta volatilidad, el umbral es m√°s alto para evitar se√±ales falsas, mientras que en momentos de baja volatilidad, el umbral disminuye para aprovechar mejor las oportunidades. Esta estrategia busca una mayor flexibilidad y adaptabilidad, permitiendo que las decisiones de inversi√≥n se ajusten a las condiciones cambiantes del mercado y potencialmente mejoren el rendimiento.

test_data_base <- test_data %>%
  select(Start, Open, predicted_price, Close, weekday) %>%
  mutate(
    predicted_return = predicted_price - Open,
    actual_return = Close - Open,
    predicted_return_pct = predicted_return / Open,
    actual_return_pct = actual_return / Open
  )

initial_investment <- 1000

estrategia1 <- test_data_base %>%
  mutate(
    theoretical_return = ifelse(predicted_return > 0, actual_return, 0),
    inv_estrategia1 = initial_investment * cumprod(1 + replace_na(theoretical_return / Open, 0))
  ) %>% select(Start, inv_estrategia1)

dias_negativos <- c("martes", "jueves")

estrategia2 <- test_data_base %>%
  mutate(
    invertir = predicted_return > 0 & !(weekday %in% dias_negativos),
    daily_return_pct = ifelse(invertir, actual_return / Open, 0),
    inv_estrategia2 = initial_investment * cumprod(1 + replace_na(daily_return_pct, 0))
  ) %>% select(Start, inv_estrategia2)

sd_return <- sd(test_data_base$actual_return, na.rm = TRUE)
threshold <- 0.5 * sd_return

estrategia3 <- test_data_base %>%
  mutate(
    invertir = predicted_return > threshold,
    theoretical_return = ifelse(invertir, actual_return, 0),
    inv_estrategia3 = initial_investment * cumprod(1 + replace_na(theoretical_return / Open, 0))
  ) %>% select(Start, inv_estrategia3)

estrategia4 <- test_data_base %>%
  mutate(
    volatility_accum = sqrt(cummean((actual_return_pct - cummean(actual_return_pct))^2)),
    threshold = 0.5 * volatility_accum,
    invertir = predicted_return_pct > threshold,
    daily_return_pct = ifelse(invertir, actual_return_pct, 0),
    inv_estrategia4 = initial_investment * cumprod(1 + replace_na(daily_return_pct, 0))
  ) %>% select(Start, inv_estrategia4)

comparativo <- estrategia1 %>%
  left_join(estrategia2, by = "Start") %>%
  left_join(estrategia3, by = "Start") %>%
  left_join(estrategia4, by = "Start") %>%
  pivot_longer(cols = starts_with("inv_"), names_to = "estrategia", values_to = "valor") %>%
  filter(!is.na(valor))

comparativo$estrategia <- recode(comparativo$estrategia,
  "inv_estrategia1" = "Estrategia 1",
  "inv_estrategia2" = "Estrategia 2 (evita martes/jueves)",
  "inv_estrategia3" = "Estrategia 3 (umbral fijo)",
  "inv_estrategia4" = "Estrategia 4 (volatilidad adaptativa)")
